# 2026-03-22 开发日志

## 今日目标

继续闭环“四通道独立工作”问题，重点收敛下面这个现象：

- 只打开 `CH0` 和 `CH2` 时，回环通信会出现异常报错
- 四个通道全打开时，`CH0/CH2` 反而能正常通信

用户明确要求排查方向偏向设备侧，不接受“必须四路都开才正常”的临时状态。

## 今日结论

今天把问题拆成了两层，并分别验证：

### 1. Host 侧是否真的一次下发了四个通道配置

已确认：

- 上位机连接后会按 `CH0..CH3` 全量下发 `SetChannelConfig`
- 即使 UI 里只显式配置了 `CH0/CH2`，`CH1/CH3` 也会收到“显式关闭”的配置
- 所以这不是“Host 漏发 `CH1/CH3` 配置”导致的

对应代码：

- `Host_PC/EverVance/src/MainForm.cs`
- `AutoSendChannelConfigsOnConnect()`
- `BuildDisconnectedChannelConfigRow()`

### 2. `STB` 极性与 `STB` 映射是两个不同问题

今天重点核对了两类证据：

#### 2.1 TJA1042 手册

资料：

- `Hardware/TJA1042.pdf`

关键结论：

- `STB = LOW -> Normal`
- `STB = HIGH -> Standby`

所以当前驱动里：

- `Normal -> GPIO 写 0`
- `Standby -> GPIO 写 1`

是正确的，**不是 `STB` 极性写反**。

#### 2.2 网表与实机对照

资料：

- `Hardware/Netlist_Schematic1_2026-03-08.tel`

网表名义关系：

- `CH2(FLEXCAN1)` 名义上对应 `U401 / CAN_1`
- `CH3(FLEXCAN2)` 名义上对应 `U403 / CAN_2`

但是今天的实机对照实验结果非常明确：

##### 方案 A：按网表名义顺序恢复 `CH2/CH3 -> STB` 映射

结果：

- 只开 `CH0/CH2` 时稳定复现：
  - `CH0 ACK Error`
  - `CH2 Bit Error`
  - `CH2 BusOff`

##### 方案 B：恢复当前这版“可工作”的 `STB` 映射

结果：

- 同样的“只开 `CH0/CH2`”场景恢复正常
- 直接 WinUSB 最小脚本可得到：
  - `CH0 TX 0x100 OK`
  - `CH0 RX 0x120 OK`
  - `CH2 TX 0x120 OK`
  - `CH2 RX 0x100 OK`
- 12 秒双通道压力结果：
  - `Err0 = 0`
  - `Err2 = 0`

因此今天最终确认的是：

- **不是 `STB` 极性反了**
- **也不是 Host 没有下发四个通道配置**
- **当前实物板上，逻辑 `CH2/CH3` 对应的有效 `STB` 控制关系，与当前仓库中网表名义顺序不一致**

这里刻意不写“原理图一定错了”，因为还存在几种可能：

- 实物板版本与仓库网表版本不一致
- 外部接口命名与网表 `CAN_1/CAN_2` 的命名口径不同
- 实板改版/飞线后，文档未同步

当前项目里，先以“实机可重复复现的行为”作为软件映射依据。

## 今日做过的关键测试

### 测试一：最小双通道回环

场景：

- `CH0 enabled = true`
- `CH2 enabled = true`
- `CH1 enabled = false`
- `CH3 enabled = false`
- `CH0/CH2` 各发一帧

在当前工作映射下，结果正常：

- `CH0 TX/RX 正常`
- `CH2 TX/RX 正常`

### 测试二：双通道 12 秒压力

场景：

- `CH0/CH2` 每 100 ms 各发一帧
- `CH1/CH3` 保持关闭

结果：

- `OkTx0 = 97`
- `OkTx2 = 98`
- `OkRx0 = 98`
- `OkRx2 = 98`
- `Err0 = 0`
- `Err2 = 0`

### 测试三：恢复网表名义顺序后复测

结果立即恶化：

- `CH0 ACK Error`
- `CH2 Bit Error`
- `CH2 BusOff`

这个对照实验是今天最关键的证据。

## 当前项目状态

### 已确认恢复

- 只开 `CH0` 和 `CH2` 时，可以独立工作
- 不需要再把四个通道全部打开
- `STB` 高低电平极性正确
- Host 连接后会全量下发四路配置

### 当前保留结论

- 当前软件需要保留“与网表名义不同的 `CH2/CH3` STB 映射”
- 这不是长期最优状态，但在当前实物板上是唯一通过实机验证的工作状态

## 后续建议

如果后续要把这件事彻底闭环，建议按下面顺序：

1. 直接测 `GPIO2_IO07` 和 `GPIO2_IO05`
   - 分别只开 `CH2`
   - 分别只开 `CH3`
   - 看哪个引脚的电平变化真正对应当前工作的总线

2. 对照实物板版本、BOM 或改版记录，确认是否存在：
   - 文档版本与实板版本不一致
   - `CAN_1/CAN_2` 命名口径变化
   - 飞线/改版未同步到仓库网表

3. 若硬件文档后续得到正式确认，再考虑把软件映射改回“文档一致”的版本

在那之前，当前代码应继续以“实机验证通过”为准。
