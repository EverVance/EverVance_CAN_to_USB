# USB 首次建链与驱动策略（2026-03-11）

## 1. 目的
- 记录 `RT1062 Bulk CAN` 在 Windows 上首次建链时的实际坑点。
- 约束后续开发、联调、量产导入时的驱动策略，避免重复踩坑。

## 2. 当前设备标识
- 设备名称：`RT1062 Bulk CAN`
- `VID/PID`：`1FC9:0135`
- 当前链路类型：`USB Vendor/Bulk + WinUSB`

## 3. 已确认的现实约束
- Windows 会按 `VID/PID[/REV]` 为设备建立独立的驱动绑定。
- 设备端若修改 `bcdDevice`，Windows 可能把它视为新修订版并重新走一次首次安装流程。
- 因此，“固件 USB 描述符变更”不只是固件问题，也会影响 PC 端首次建链。

## 4. 不建议作为默认方案的做法
- 未签名 INF

原因：
- 需要管理员权限。
- 易被企业电脑的驱动签名策略阻拦。
- 维护性差，不适合作为量产或售后默认路径。
- 一旦设备修订号变化，可能再次触发人工装驱动。

## 5. 推荐优先级
1. 量产默认：设备端通过 Microsoft OS Descriptors 让 Windows 自动绑定 `WinUSB`。
2. 次优：提供已签名驱动包（INF/CAT），首次接入新电脑时安装一次。
3. 临时研发：`Zadig` 或测试签名模式，仅用于实验室恢复链路，不纳入量产 SOP。

## 6. 首次建链判定标准
以下 3 条同时成立，才算“首次建链成功”：
1. 设备管理器中出现 `RT1062 Bulk CAN`，状态 `OK`
2. Host_PC 能成功 `Open()` WinUSB 传输层
3. Host_PC 与设备至少完成一次控制命令往返，如 `GetDeviceInfo` / `Heartbeat`

## 7. 必须留痕的场景
- 新电脑首次接入
- 变更 `VID/PID`
- 变更 `bcdDevice`
- 变更 `DeviceInterfaceGUID(s)`
- 变更 Microsoft OS Descriptor 相关实现
- 从“自动绑定”切换到“人工装驱动”或反向切换

留痕至少包含：
- 电脑名
- 操作者
- 固件版本
- `VID/PID/REV`
- 驱动方案
- 是否需管理员权限
- 是否适用于量产

## 8. 当前阶段建议
- 短期目标：先把“Windows 自动绑定 WinUSB + Host_PC 稳定打开”收敛到无需人工介入。
- 中期目标：若自动绑定在目标 Windows 环境仍不稳定，再补“已签名驱动包”作为正式备份方案。
- 明确不建议把“未签名 INF 手工安装”写进量产指导。
